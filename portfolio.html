Hero Trial




Click here to play!

Description:

In the game Hero Trial, the player is a hero-in-training who must clear a trial of dangerous monsters in order to become a true legend. When the player is able to defeat a slime monster, they earn a point. Randomness is used to determine when the slime will spawn and how fast the slime will move. Once the player reaches 15 points, the slime will being to spawn much faster and move much quicker. The game is won when the player earns 30 points. The game is lost when the player loses 3 of their lives.

Self-Reflection:

My lonely childhood consisted of me longing to be an admired hero who would save those in trouble. This inspired my partner and me to create a game where the player would be able to become a true hero if they clear the daunting task ahead of them. Also, I have an app on my phone called Will Hero that also gave my group some inspiration for the game. One successful point in the development process was making our own characters from scratch. We were able to design our own character, walking animation, standing animation, and attack animation. Another successful point is getting the home screen to work smoothly. Two obstacles that arose included allowing both of us to work on the game at the same time and finding a way to implement a score system into the game. We overcame the first obstacle by having one of us create a remix of the game and working on the code in a second account. However, this proved to be inefficient as we would use different algorithms for different characters, which made one of our characters (the knight) unplayable. We overcame the second obstacle by making it so if the player kills a slime, they would earn one point. If my parter and I had more time, we would fix the game so the knight character would be playable, add different kinds of monsters, and add powerups.

Helpful Algorithm:

This is an algorithm because it calculates the x-position and y-position of the wizard to determine where the fireball will spawn. It also sets the rule that it will continue until it reaches the edge of the screen or until it touches the slime. Essentially, the algorithm makes it so when the space bar is pressed, the wizard will shoot a fireball that will continue to fly until it hits a slime or reaches the edge of the screen.

The Heroic Princess?

Storyline:

You're a princess who is being held hostage in a strange tower by a terrifying monster. A knight in rusty armor bravely barges in to try to save you, but his cowardice and patheticness renders him useless. Now, it's up to you to find a way to escape from the beast.

Self-Reflection:

First, I brainstormed ideas for my story, placing it in a flow chart. Afterwards, I organized the scenes into functions. Now, I needed to implement a system so the user can input a decision at a decision point. I tried to use an or boolean operator in the while loop, but it didn't work properly. While inputting "1" successfully ran option 1, inputting "2" resulted as invalid. Since the test failed, I had to refine the code and settled on using the in function with the string "12" for the condition instead. This successfully allowed the user to choose either option by inputting "1" or "2." Then, I made a typing function by importing sleep to create a delay between each character appearing in the console. In the first test, the text appeared too fast, making it difficult for the user to follow. I adjusted the delay accordingly, but this time, the delay was too long, making the story drag out. I kept refining the code until I reached a happy medium that was a comfortable pace for the user to read.

Organizing scenes of my story into functions enables myself to later call on the function to play out the scene without worrying about how the function works. This way, in order to play out a scene that I want, all I have to do is simply call on the function and not worry about coding out the scene. This is a a form of procedural abstraction, which is implementing functions that when called on computes an algorithm or process.

One story I particularly liked was Batman's Trilogy because the results were sometimes completely unexpected, random, and outright ridiculous. This made the story stand out from the rest.
